## üõ†Ô∏è Your Action Plan as IT Support

Your next steps should focus on gathering information and preparing the physical and logical network infrastructure.

### Step 1: Initial Investigation (Ask Questions!)

You must clarify the technical requirements before running any cables.

1.  **Which two machines exactly?** (e.g., Is it the **Sinto FBO III A(N)** Molding Machine and the **Multicooler** Sand Plant?)
2.  **What port/interface are we connecting to?** (The PLC unit usually has an RJ-45 Ethernet port, but confirm this.)
3.  **What is the objective?** (E.g., "We need to connect the PLC on the Sinto machine so the maintenance team can remotely monitor the production counter and cycle time.")
4.  **Who needs access?** (Which team/engineer needs to use the network connection?)

### Step 2: Physical Network Preparation

* **Cable Run:** Plan the path for running two **industrial-grade Ethernet cables** (ideally **shielded/SSTP**) from the machine control cabinets to the nearest network switch. The foundry environment requires durable cabling.
* **Switch Ports:** Ensure you have two available, working ports on a network switch near the machines.

### Step 3: Logical Network Preparation

This is the core IT job:

* **IP Allocation:** Allocate two **Static IP Addresses** for the two PLC/Control units. (Example: `192.168.10.150` and `192.168.10.151`).
    * ***Crucial Note:*** PLCs often require IPs in a specific range defined by the OT/Automation engineer. **Always coordinate the IP settings with the automation team.**
* **Security:** If your network is segmented, ensure the switch ports are configured for the correct **VLAN** (usually an OT/Industrial Control System VLAN) to limit traffic and enhance security.
* **Firewall Rules:** If the "main server" (likely a SCADA or Data Historian server) is in a different network segment, prepare to create firewall rules to allow the PLC-specific traffic (like Modbus TCP on port 502) to reach that server.

### Step 4: Execution & Handover

1.  **Install Cables:** Run and terminate the two Ethernet cables.
2.  **Configuration:** Set the static IP, subnet mask, and gateway on the PLC's network interface (this is often done by the automation engineer, but you provide the details).
3.  **Test:** Ping the new IP addresses from your network to confirm basic connectivity.
